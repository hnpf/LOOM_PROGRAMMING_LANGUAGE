Loom Language Reference

A comprehensive guide to Loom's syntax, types, and core features.

Fundamental Types

Type Description Example
int 64-bit signed integer 42, -10
float 64-bit floating point 3.14, -0.001
str UTF-8 String (supports {interpolation}) "Loom", "Result: {val}"
char Single Unicode character 'a', 'λ'
bool Boolean logic true, false
List Dynamic array of elements [1, 2, 3]
Result Error handling variant Ok(val), Err("fail")
Option Optional value variant Some(val), None

Traits & Weave

Traits define shared behavior, and weave implements them for specific frames.

trait Drawable {
    act draw(self)
}

weave Drawable into Player {
    act draw(self) {
        print("Drawing {self.name}")
    }
}

Expressive Comments

Loom uses semantic comment prefixes for documentation and scanning.

Prefix Meaning
//*Logic block/section header
//!Urgent/Critical warning
//?Query/Debug thought
//.Metadata (Author, Version)
//;Legal/Copyright

Variable Bindings

Loom supports mutable bindings with let and immutable constants with const.

let mutable_var = 10
const PI = 3.14159

Control Flow

Conditionals (if/else)

if x > 10 { print("Large") } else { print("Small") }

If Let (Pattern Binding)

Use if let to safely extract values from Results or Options.

if let Ok(data) = read("config.json") {
    print("Read {data.len()} bytes")
}

Loops

Loom provides loop, while, and for loops.

loop { 
    if done { break }
}

while i < 10 { 
    i = i + 1 
}

for i in 0..10 { 
    print(i) 
}

for item in list { 
    print(item) 
}

Pattern Matching (when)

Pattern matching is the idiomatic way to handle Results and Options.

when result {
    Ok(val)  => print("Success: {val}"),
    Err(msg) => print("Error: {msg}"),
    _        => print("Catch-all"),
}

Frames & Methods

Frames define data structures, and bind defines behavior.

frame Point { x: int, y: int }

bind Point {
    act move(self, dx, dy) {
        self.x = self.x + dx
        self.y = self.y + dy
    }
}

Linking Components

Loom excels at linking these pieces. You can use Frames inside Lists, spawn methods of Frames, or use Shell output to populate Frame fields.

let results = []
let handle = spawn { $ "uptime" }
let out = handle.await()
if out.status == 0 {
    results.push(out.stdout)
}
