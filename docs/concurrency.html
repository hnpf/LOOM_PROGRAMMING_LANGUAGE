Concurrency in Loom

Loom makes concurrency simple and safe with lightweight tasks. Tasks are managed by the runtime and provide structured concurrency primitives.

Spawning Tasks

Use the spawn keyword to start a new task. It takes a block of code and returns a Task handle immediately.

let handle = spawn {
    print("Background task running...")
    return 42
}

Awaiting Results

To synchronize and get the result of a task, call the .await() method on the handle.

let result = handle.await()

when result {
    Ok(val) => print("Task completed: {val}"),
    Err(e)  => print("Task failed: {e}"),
}

The result is always wrapped in a Result type (Ok or Err).

Parallel Processing

Tasks are most powerful when combined with list map for parallel data processing.

let ids = [1, 2, 3]
let tasks = ids.map(act(id) {
    spawn { process(id) }
})

let results = tasks.map(act(t) { t.await() })
