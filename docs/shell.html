Shell Integration in Loom

Loom is designed to be a "shell-aware" language. This means running system commands isn't a library call—it's a core language feature.

The Shell Operator ($)

Any string prefixed with the $ operator will be executed as a system command in the sh environment.

let res = $ "echo Hello Shell"
print(res.stdout)

String Interpolation

You can link Loom variables into your shell commands using curly braces {}. Loom handles the formatting automatically. You can also interpolate fields directly from frames or objects.

let dir = "/home/user"
let res = $ "ls -l {dir}"
print(res.stdout)

// Interpolating fields directly
let config = { port: 8080, host: "localhost" }
print("Connecting to {config.host}:{config.port}")

The ShellOutput Frame

Every $ operator returns a built-in ShellOutput object with the following fields:

Field Type Description
stdout str Standard output of the command (trimmed).
stderr str Standard error of the command.
status int The exit code (0 for success).

Linking Shell with Concurrency

The most powerful use case for Loom is spawning shell commands in parallel.

act check_service(name) {
    let res = $ "systemctl is-active {name}"
    return res.stdout == "active"
}

act main() {
    let services = ["nginx", "postgresql", "docker"]
    let results = services.map(act(s) {
        spawn { check_service(s) }
    })

    for r in results {
        when r.await() {
            Ok(active) => print("Service active: {active}"),
            _          => print("Failed check"),
        }
    }
}

main()
