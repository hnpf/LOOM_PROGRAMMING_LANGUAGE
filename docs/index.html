Welcome to Loom (v0.1)

Loom is a modern, concurrent programming language designed for crafting developer tools and system automation. It fills the gap between the rapid iteration of shell scripts and the safety of high-level programming.

High Performance 🚀

Loom's weave compiler offers up to 50x speedup over interpreted code. Benchmarked for shell spawning, string handling, and recursion.

See the results →
The Loom Philosophy: We believe that the terminal is a first-class citizen. Loom brings structure to your shell and simplicity to your parallel tasks.

Why Loom?

Shell-First

Run shell commands directly with the $ operator. No more subprocess.run or complex piping. Catch output, status, and stderr effortlessly.

Structured Concurrency

Native spawn and await blocks that make background tasks as easy as a function call. Built-in Task management ensures your parallel work is safe.

Self-Documenting

Our unique Expressive Comment system (//*, //!, //?) allows you to generate professional documentation automatically while keeping your code clean.

Trait-Based Design

Decouple data from behavior with trait and weave. Add new functionality to existing types without modifying their original definition.

Ghost Imports

Import logic directly from Markdown with pull "logic.md". Your documentation is your source of truth. No more stale READMEs.

Safety Blocks

Resource-aware execution with safety(mem: 128MB) blocks. Protect your hardware and get automatic crash reports for future debugging.

A Quick Taste

A concurrent system check with auto-documentation markers:

//; system health check | mit license
//* diagnostic logic

act main() {
    //? should we add a timeout to this task?
    let handle = spawn { $ "grep -c 'ERROR' app.log" } //! potential heavy I/O
    
    print("Checking logs in the background...")
    
    when handle.await() {
        Ok(res) => print("Found {res.stdout} errors."),
        Err(e)  => print("Log check failed: {e}"),
    }
}

main()
